# Time University カリキュラム（教育設計フレームワーク準拠版）

## 採用フレームワーク

| フレームワーク | 適用範囲 | 役割 |
|---------------|---------|------|
| ブルームのタキソノミー | レベル設計・学習目標・試験設計 | 「何ができたら合格か」を6段階で定義 |
| メリルの第一原理 | 各講義の構成 | 1コマの授業を5要素で設計 |
| コルブの経験学習サイクル | 実技課題・カリキュラム全体の循環 | 経験→省察→概念化→実験のサイクルを回す |

補助フレームワーク:
- コンストラクティブ・アラインメント（学習目標-学習活動-評価の整合性）
- 反転学習（事前学習=無料コンテンツ、対面=ワークショップ）
- 70:20:10モデル（経験70%:社会20%:公式10%の学習配分）

---

## 全体構造

```
TMBOK（知識体系）
  |
  ├── L1カリキュラム（記憶→理解→応用）
  |     ├── 基礎知識科目群（オンライン自学）
  |     ├── 実践科目群（ワークショップ+実技）
  |     └── 認定試験
  |
  ├── L2カリキュラム（応用→分析→評価）
  |     ├── 応用知識科目群（オンライン自学）
  |     ├── 実践科目群（ワークショップ+実技）
  |     └── 認定試験
  |
  └── L3カリキュラム（評価→創造）
        ├── 専門知識科目群（オンライン自学）
        ├── 実践科目群（研修設計実習）
        └── 認定試験
```

### 学習形態（反転学習モデル）

```
[事前学習] テキスト・動画・Podcast（Time University無料コンテンツ）
     |
     ↓  ※ 知識の習得はここで完了させる
     |
[対面/同期] ワークショップ・ディスカッション・実践演習
     |
     ↓  ※ 対面の時間は「応用・分析・議論」に全振り
     |
[実技課題] 業務での実践 → レポート提出
     |
     ↓  ※ コルブのサイクルを回す
     |
[認定試験]
```

## TMBOK: Time Management Body of Knowledge

5領域 x 3層（概念→エビデンス→実践）で構成。

| 領域 | 概念（何を知るか） | エビデンス（なぜそう言えるか） | 実践（どうやるか） |
|------|-------------------|----------------------------|-------------------|
| 1. 時間哲学 | 有限性、時間貧困、緊急性バイアス、生産性の逆説 | Burkeman, Pencavel, Miura, Zhu, Whillans | 時間の優先順位設計、「やらないことリスト」作成 |
| 2. 時間科学 | 注意残余、47秒問題、フロー状態、クロノタイプ | Leroy, Mark, Alameda, Rosen, 島津 | Ready-to-Resume Plan、集中環境設計、中断ログ |
| 3. 時間経営 | GTD/Pomodoro/Slow Productivity、週4日制、ハイブリッドワーク、チーム時間設計 | Aeon, Biwer, Schor, Bloom, Shangguan | フレームワーク選択、制度設計提案、チーム時間監査 |
| 4. 時間工学 | 時間記録の原則、データリテラシー、改善サイクル、時間監査 | Drucker, J-STAGE, Microsoft NFW | TimeCrowdによる記録・分析・可視化・改善 |
| 5. 時間未来学 | AI生産性、ギザギザのフロンティア、仕事の膨張、時間再配分 | Noy, Dell'Acqua, Lee, HBR | AI活用戦略設計、再配分計画立案 |

## Level 1 カリキュラム

### 到達レベル: ブルームのタキソノミー 記憶→理解→応用

「時間管理の基本概念を理解し、自分の時間を記録・分析・改善できる」

### 学習目標（SMART形式 x タキソノミー動詞）

| # | 学習目標 | タキソノミー段階 | 評価方法 |
|---|---------|---------------|---------|
| L1-01 | ドラッカーの時間管理3ステップ（記録→整理→まとめる）を**説明**できる | 理解 | 知識試験 |
| L1-02 | 時間の有限性（4,000週）と生産性の逆説（55時間の壁）を**説明**できる | 理解 | 知識試験 |
| L1-03 | 注意残余の概念を**定義**し、47秒問題の原因を**説明**できる | 記憶・理解 | 知識試験 |
| L1-04 | Mere Urgency Effect（緊急性バイアス）を**説明**し、自分の業務で該当するタスクを**識別**できる | 理解・応用 | 知識試験+実技 |
| L1-05 | GTD / Pomodoro / Slow Productivity の違いを**比較**し、自分に適した手法を**選択**できる | 理解 | 知識試験 |
| L1-06 | 時間管理がパフォーマンスよりウェルビーイングに効く理由を**説明**できる | 理解 | 知識試験 |
| L1-07 | Ready-to-Resume Planを日常業務で**実践**できる | 応用 | 実技課題 |
| L1-08 | 自分の業務時間を1週間以上リアルタイムで**記録**できる | 応用 | 実技課題 |
| L1-09 | 記録データから「思考/作業/中断/会議」の比率を**算出**できる | 応用 | 実技課題 |
| L1-10 | データに基づく改善ポイントを1つ以上**特定**し、改善アクションを**提案**できる | 応用 | 実技課題 |

### 科目構成

#### 基礎知識科目（事前学習 -- オンライン自学）

| 科目コード | 科目名 | TMBOK領域 | 時間 | 形式 | 学習目標 |
|-----------|--------|----------|------|------|---------|
| L1-K1 | Know Thy Time -- ドラッカーの時間管理思想 | 4. 時間工学 | 60分 | 動画+テキスト | L1-01 |
| L1-K2 | 時間の有限性と生産性の逆説 | 1. 時間哲学 | 60分 | 動画+テキスト | L1-02 |
| L1-K3 | 注意残余と47秒問題 | 2. 時間科学 | 60分 | 動画+テキスト | L1-03 |
| L1-K4 | 緊急性バイアスと時間貧困 | 1. 時間哲学 | 45分 | 動画+テキスト | L1-04 |
| L1-K5 | 時間管理フレームワーク比較 | 3. 時間経営 | 60分 | 動画+テキスト | L1-05, L1-06 |
| L1-K6 | 集中環境の設計とReady-to-Resume Plan | 2. 時間科学 | 45分 | 動画+テキスト | L1-07 |
| | **合計** | | **5.5時間** | | |

各科目の構成（メリルの第一原理に準拠）:

```
1. Problem（問題提起）    -- 5分  「あなたの"忙しい"の正体は？」
2. Activation（活性化）   -- 5分  「普段の時間の使い方を思い出してください」
3. Demonstration（例示）  -- 25分  学術データ・事例の提示
4. Application（応用）    -- 15分  ミニ演習・セルフチェック
5. Integration（統合）    -- 10分  「自分の業務にどう適用するか」振り返り
```

#### 実践科目（対面/同期 -- ワークショップ）

| 科目コード | 科目名 | TMBOK領域 | 時間 | 形式 | 学習目標 |
|-----------|--------|----------|------|------|---------|
| L1-P1 | 時間記録ワークショップ | 4. 時間工学 | 90分 | 対面 | L1-08 |
| L1-P2 | 時間データ分析ワークショップ | 4. 時間工学 | 90分 | 対面 | L1-09, L1-10 |
| | **合計** | | **3時間** | | |

各ワークショップの構成（コルブの経験学習サイクルに準拠）:

**L1-P1: 時間記録ワークショップ（90分）**
```
1. 具体的経験（30分）  -- TimeCrowdで実際に時間を記録する（その場で30分間）
2. 内省的観察（20分）  -- 30分間の記録を見て気づいたことを書き出す
3. 抽象的概念化（20分） -- ドラッカーの「記録の3ステップ」と結びつける
4. 能動的実験（20分）  -- 翌1週間の記録計画を立てる
```

**L1-P2: 時間データ分析ワークショップ（90分）**
```
1. 具体的経験（10分）  -- 1週間の記録データを持参。まず自分で眺める
2. 内省的観察（25分）  -- データからパターンを3つ見つける（個人ワーク）
3. 抽象的概念化（25分） -- グループで共有。「思考/作業/中断/会議」の分類を学ぶ
4. 能動的実験（30分）  -- 改善ポイントの特定+改善アクションの設計+レポート骨子作成
```

### L1 認定試験

| 区分 | 形式 | 配点 | タキソノミー段階 | 合格基準 |
|------|------|------|---------------|---------|
| 知識試験 | オンライン選択式25問+短答式5問（60分） | 50点 | 記憶・理解 | 35点以上 |
| 実技課題 | 1週間の時間記録+分析レポート | 50点 | 応用 | 35点以上 |
| **合計** | | **100点** | | **70点以上** |

※ 70:20:10モデルに基づき、知識50%:実技50%の配分に修正（旧版の70%:30%から変更）

### L1 学習パス全体（コンストラクティブ・アラインメント検証済み）

```
Week 1-2: 基礎知識科目 L1-K1〜K6（オンライン自学、計5.5時間）
    |
Week 3:   実践ワークショップ L1-P1（時間記録、90分）
    |
Week 3-4: 1週間の時間記録（業務内で実施）
    |
Week 4:   実践ワークショップ L1-P2（データ分析、90分）
    |
Week 4-5: 分析レポート作成・提出
    |
Week 5-6: 知識試験受験（オンライン、任意のタイミング）
    |
    → 合格: タイムマネジメントスペシャリスト L1 認定
```

**総学習時間: 約12-15時間**（自学5.5h + WS3h + 記録・分析4-6h + 試験準備1h）

## カリキュラムマトリクス（コンストラクティブ・アラインメント検証）

各TMBOK領域が、どのレベルでどの深さまで扱われるかの対応表。

| TMBOK領域 | L1 | L2 | L3 |
|----------|-----|-----|-----|
| 1. 時間哲学 | I/R（導入・強化） | -- | A（習熟前提） |
| 2. 時間科学 | I/R（導入・強化） | -- | A（習熟前提） |
| 3. 時間経営 | I（導入のみ） | R/A（強化・習熟） | A（習熟前提） |
| 4. 時間工学 | I（導入のみ） | R/A（強化・習熟） | A（習熟前提） |
| 5. 時間未来学 | -- | I/R（導入・強化） | A（習熟前提） |
| 教授法 | -- | -- | I/R/A（全段階） |

I = Introduced（導入）、R = Reinforced（強化）、A = Applied at mastery（習熟レベルで適用）
