## 第1回: 時間の正体を知る
**サブタイトル**: 「忙しい」の解剖 -- なぜ思考時間が消えるのか

### 学習目標
1. 「時間が足りない」という感覚の正体が、絶対量の不足ではなく配分の構造問題であることを理解する
2. 長時間労働の収穫逓減を数値で把握する
3. 「緊急性バイアス」の存在を知り、自分の時間配分を疑う視点を持つ

### 主要概念
- **4,000週の有限性**: 人生80年=約4,000週（Burkeman）
- **55時間の壁**: 週55時間以上はゼロ生産性（Pencavel 2015）
- **時間貧困**: 客観的な時間ではなく主観的な「時間が足りない感覚」（Miura 2025, Whillans）
- **Mere Urgency Effect**: 緊急だが重要でないタスクを優先してしまうバイアス（Zhu 2018）

### 現実世界の事例
- 日本マイクロソフト2019年の週4日トライアル: 金曜全休にして生産性40%向上
- Pencavelの研究: 第一次大戦中の英国軍需工場データから「週70時間と55時間の生産量が同じ」
- Whillansの研究: 時間節約サービスにお金を使う人は、より幸福

### ディスカッション用の問い
1. 「先週の業務時間のうち、本当に"考えていた"時間は何時間ありましたか？」
2. 「あなたの組織で"長く働く人が評価される"という空気はありますか？ それはなぜ存続しているのでしょうか？」
3. 「今日のTo-Doリストの中で、"緊急だが重要でないもの"はどれですか？」

### 60分の時間配分
| 時間 | 内容 | 形式 |
|------|------|------|
| 0-5分 | 問いかけ: 先週「考えていた」時間は？ 参加者に挙手/チャット | インタラクション |
| 5-15分 | 4,000週の有限性 + 55時間の壁（データ提示） | 講義 |
| 15-25分 | 時間貧困の科学 -- Miura尺度 + Whillans研究 | 講義 |
| 25-35分 | Mere Urgency Effect: 重要と緊急の取り違え | 講義+ワーク |
| 35-50分 | ディスカッション: 自分の時間配分を棚卸し | グループワーク |
| 50-55分 | 日本マイクロソフト事例 -- 構造を変えたら結果が変わった | 講義 |
| 55-60分 | まとめ: 問題は「時間の量」ではなく「時間の構造」 + 次回予告 | 講義 |

### 産学連携ポイント
- **ゲスト候補**: 一川誠教授（千葉大） -- 「時間デザイン」の概念から導入
- **データ連携**: TimeCrowdの匿名化データで「日本のビジネスパーソンの平均的な時間配分」を提示

## 第2回: Deep Workと集中の科学
**サブタイトル**: 注意残余とタスクスイッチング -- なぜ「ちょっと確認」が危険なのか

### 学習目標
1. 注意残余（attention residue）のメカニズムを理解し、タスク切替のコストを数値で把握する
2. フロー状態の神経科学的条件を知る
3. Ready-to-Resume Planを実践できるようになる

### 主要概念
- **注意残余**: タスクAからBに移った後もAへの注意が残り続ける現象（Leroy 2009）
- **47秒問題**: 現代人の注意持続時間の実態。自己中断が49%（Gloria Mark 2023）
- **フロー状態**: 前頭前皮質の明示的制御が低減する「一過性低前頭化」（Rosen 2024）
- **Ready-to-Resume Plan**: 中断前に「どこまでやったか」を書き留める手法（Leroy 2018）

### 現実世界の事例
- Gloria Markの20年追跡: 2004年に2.5分だった注意持続時間が2023年には47秒に
- 通知の認知コスト: スマートフォン通知の頻度がスクリーンタイム総量より注意散漫の予測因子
- Microsoft 2024: 1日275回の中断、会議の60%がアドホック

### ディスカッション用の問い
1. 「昨日、最も集中できた時間帯はいつでしたか？ その前後に何をしていましたか？」
2. 「あなたの職場で"中断されない1時間"を確保するには、どんな障壁がありますか？」
3. 「Ready-to-Resume Planを今日から使うとしたら、どの場面で使いますか？」

### 60分の時間配分
| 時間 | 内容 | 形式 |
|------|------|------|
| 0-3分 | 前回の振り返り + 今回のテーマ | 講義 |
| 3-15分 | 注意残余の科学 -- Leroy原著 + 47秒問題 | 講義 |
| 15-22分 | 通知の隠れたコスト + 1日275回の中断 | データ提示 |
| 22-32分 | フロー状態の神経科学 -- 脳が最も機能する条件 | 講義 |
| 32-40分 | Ready-to-Resume Plan: 実践ワーク | 個人ワーク |
| 40-50分 | ディスカッション: 自分の「中断パターン」を特定する | グループワーク |
| 50-57分 | Deep Workの4つのルール（Newport）-- 環境設計の具体策 | 講義 |
| 57-60分 | まとめ + 1週間の実践課題: 「中断ログをつけてみる」 | 講義 |

### 産学連携ポイント
- **ゲスト候補**: 河原純一郎教授（北大） -- 注意の認知メカニズム / 伊達洋駆氏 -- 注意残余のビジネス翻訳
- **実践課題**: TimeCrowdで1週間の「中断パターン」を記録してもらい、第4回で分析

## 第3回: 時間管理フレームワーク比較
**サブタイトル**: GTD / Pomodoro / Slow Productivity -- 何を選ぶか、どう組み合わせるか

### 学習目標
1. 3つの主要フレームワーク（GTD, Pomodoro, Slow Productivity）の思想と使い分けを理解する
2. 時間管理の効果に関するメタ分析の知見を把握する
3. 自分の業務特性に合ったフレームワークを選択できるようになる

### 主要概念
- **GTDの5ステップ**: Capture, Clarify, Organize, Reflect, Engage（Allen）
- **ポモドーロの科学的検証**: 組織的休憩が自己調整休憩より効果的（Biwer 2023）
- **Slow Productivityの3原則**: Do Fewer Things / Work at a Natural Pace / Obsess Over Quality（Newport 2024）
- **メタ分析の知見**: 時間管理は「パフォーマンス」より「ウェルビーイング」に効く（Aeon 2021）

### 現実世界の事例
- Aeonメタ分析: 158研究が示す「時間管理は生活満足度に最も強い効果」
- Biwer実験: ポモドーロ式24分+6分休憩が、自由な休憩取得より集中力・モチベーションで優位
- Pedersen (2024): 週4日制企業がポモドーロを全社導入 -- 個人の生産性と集団のバランスを両立
- Schor UK週4日制: 100%給与+80%労働時間で100%生産性維持。92%の企業が継続

### ディスカッション用の問い
1. 「あなたの業務は"タスクが細かく多い"タイプですか？ それとも"1つの大きな仕事に集中する"タイプですか？」
2. 「GTD/Pomodoro/Slow Productivity、今の自分に最も合いそうなのはどれですか？ なぜ？」
3. 「もし週4日制を提案するなら、上司をどう説得しますか？」

### 60分の時間配分
| 時間 | 内容 | 形式 |
|------|------|------|
| 0-3分 | 前回の振り返り + 中断ログの共有 | インタラクション |
| 3-12分 | メタ分析の知見: 時間管理は何に効くのか（Aeon 2021） | 講義 |
| 12-22分 | GTDの構造と思想 + Slow Productivityとの対比 | 講義 |
| 22-30分 | ポモドーロの科学的検証（Biwer 2023） | 講義 |
| 30-38分 | 3フレームワーク比較ワーク: 自分の業務特性に合うものを選ぶ | 個人ワーク |
| 38-48分 | 組織の時間設計: 週4日制（Schor）+ ハイブリッド（Bloom）の実証 | 講義 |
| 48-53分 | フレキシブルワークの逆説（Kotynkova）-- 制度だけでは解決しない | 講義 |
| 53-58分 | ディスカッション: 「週4日制を上司に提案するなら？」 | グループワーク |
| 58-60分 | まとめ + 次回予告: 「次回はデータで検証します」 | 講義 |

### 産学連携ポイント

## 第4回: 時間データで見える化する
**サブタイトル**: 計測から始まる改善 -- 時間計測の実践とデータ活用

### 学習目標
1. 主観的な時間認識と客観的なデータの乖離を認識する
2. 時間データがチームの生産性改善にどう使われるかを具体例で理解する
3. 自分の時間データを読み解き、改善ポイントを特定できるようになる

### 主要概念
- **主観と客観の乖離**: 自分では集中しているつもりが47秒で中断（Mark）
- **客観的時間把握の効果**: 客観的方法を使う労働者は生産性が高い（J-STAGE 2026）
- **チーム内時間再配分**: 生産性向上の60%はチーム内再配分による（Shangguan 2025）
- **計測→可視化→再配分サイクル**: データに基づく意思決定プロセス

### 現実世界の事例
- Shangguan (2025): 5人チームのトップ貢献者がチーム労働時間の60%超を投入 -- データで初めて見えた偏り
- TimeCrowd導入企業の事例: 時間計測データから業務プロセスを再設計
- 第2回の実践課題で参加者が記録した「中断ログ」の分析

### ディスカッション用の問い
1. 「自分の先週のカレンダーを見て、"思考時間"と"作業時間"の比率はどうでしたか？」
2. 「あなたのチームで"見えない時間の偏り"はありませんか？」
3. 「もし時間データを経営会議で報告するなら、何を見せますか？」

### 60分の時間配分
| 時間 | 内容 | 形式 |
|------|------|------|
| 0-5分 | 中断ログの結果共有 -- 参加者の実データから気づきを引き出す | インタラクション |
| 5-15分 | 主観と客観の乖離: 47秒問題 + J-STAGE 2026の知見 | 講義 |
| 15-25分 | チーム内時間再配分の威力（Shangguan 2025） | 講義 |
| 25-35分 | TimeCrowdの実務事例: データに基づく意思決定 | ケーススタディ |
| 35-45分 | ワーク: 自分の先週のカレンダーを「思考/作業/中断」に分類する | 個人ワーク |
| 45-53分 | ディスカッション: チームの「見えない時間の偏り」 | グループワーク |
| 53-58分 | クロノタイプ: 「いつ」働くかも設計対象 | 講義 |
| 58-60分 | まとめ + 次回予告: 「最終回はAI時代の時間設計」 | 講義 |

## 第5回: AI時代の時間再配分
**サブタイトル**: 削減した時間を「何に使うか」を設計する

### 学習目標
1. AIツールによる時間圧縮の実証データを把握する
2. 時間節約の「裏側」（批判的思考の低下、仕事の膨張）を認識する
3. 「時間を節約すること」と「時間を設計すること」の違いを自分の言葉で説明できるようになる

### 主要概念
- **AI生産性の実証**: ライティング40%短縮（Noy）、コンサル25%高速化（Dell'Acqua）
- **ギザギザの技術フロンティア**: AIが得意/不得意な領域は直感に反する（Dell'Acqua 2023）
- **批判的思考の低下リスク**: AIへの信頼が高いほど思考しなくなる（Lee 2025）
- **仕事の膨張パラドックス**: 浮いた時間が余暇でなく仕事に吸収される（HBR 2025）
- **時間設計 vs 時間節約**: 効率化だけでは問題は解決しない（Burkeman）

### 現実世界の事例
- Noy & Zhang (2023): MITの444名実験。ChatGPTでライティング時間40%短縮
- Dell'Acqua et al. (2023): BCG 758名。「ケンタウロス型」と「サイボーグ型」の2つのAI活用パターン
- Brynjolfsson et al. (2025): 5,179名。新人2ヶ月=ベテラン6ヶ月の生産性
- 元記事のNotebookLMワークフロー: カリキュラム構築が1サマー→1週間に
- HBR (2025): AIで浮いた時間を「もっと多くの仕事」に使ってしまう実態

### ディスカッション用の問い
1. 「もしAIで毎日2時間浮いたら、その時間を何に使いますか？」
2. 「あなたの業務で"AIに任せるべきこと"と"人間が考えるべきこと"の境界線はどこですか？」
3. 「5回の講義を通じて、明日から変える"1つのこと"は何ですか？」

### 60分の時間配分
| 時間 | 内容 | 形式 |
|------|------|------|
| 0-3分 | 4回分の振り返り: 有限性→注意→構造→計測 | 講義 |
| 3-15分 | AI生産性の実証データ: 3つの大規模実験（Noy, Dell'Acqua, Brynjolfsson） | 講義 |
| 15-22分 | NotebookLMワークフローのデモ: 時間圧縮の具体例 | デモ |
| 22-30分 | 警告1: 批判的思考の低下（Lee 2025 CHI） | 講義 |
| 30-37分 | 警告2: 仕事の膨張パラドックス（HBR 2025）+ Burkeman | 講義 |
| 37-48分 | ディスカッション: 「2時間浮いたら何に使うか？」 | グループワーク |
| 48-55分 | 全体総括: 「時間を節約すること」と「時間を設計すること」の違い | 講義 |
| 55-60分 | 締めの言葉 + 「削減された時間を何に使うかは、使う人間が決める」 | 講義 |

### 産学連携ポイント
