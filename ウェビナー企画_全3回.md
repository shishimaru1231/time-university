# ドラッカーから学ぶ、タイムマネジメントの TimeCrowd 実践ワークショップ（全3回）

対象: 既存ユーザー + 新規ユーザー
形式: オンラインウェビナー
各回60分想定
スケジュール: 第1回 4月 or 5月 / 第2回 5月 / 第3回 6月
各回3回送付（集客メール）

---

## 全体構成 -- ドラッカーの3ステップに対応

ドラッカー『経営者の条件』第2章「汝の時間を知れ」の3ステップ:

```
Step 1: 時間を記録する   → 第1回
Step 2: 時間を整理する   → 第2回
Step 3: 時間をまとめる   → 第3回
```

### 各回の共通構成

```
1. ドラッカーの考え方を整理（10分）
2. 和田としての解釈（10分）
3. TimeCrowdを使ってできる実践的な活用ケースへの落とし込み（30分）
4. Q&A / まとめ（10分）
```

## 第1回: 時間を記録する -- なぜ時間の使い方を診断する必要があるのか

### ドラッカーの3ステップにおける位置づけ
> 「成果をあげる者は仕事からスタートしない。時間からスタートする。計画からもスタートしない。時間が何にとられているかを明らかにすることからスタートする」

Step 1 = まず記録する。記憶ではなく記録で。

### 1. ドラッカーの考え方を整理（10分）

- ドラッカーが「時間」を最も希少な資源と位置づけた理由
  - "Time is the scarcest resource, and unless it is managed, nothing else can be managed."
  - お金は借りられる、人は雇える、しかし時間だけは増やせない
- 「記憶ではなく記録で」-- なぜ人間は自分の時間の使い方を正確に把握できないのか
  - ドラッカー: 実際に記録してみると、自分の認識と現実が大きくずれていることに気づく
- 3ステップの全体像を紹介（今日は Step 1）

### 2. 和田としての解釈（10分）

- 時間管理の重要性（時間の希少性）
  - 現代のデータ: 注意持続時間が47秒に短縮（Gloria Mark, 2023）
  - 週55時間以上の労働は生産性ゼロ（Pencavel, 2015）
  - ドラッカーが58年前に言っていたことが、今のデータで裏付けられている
- 時間の使い方を診断する必要がある理由
  - 「忙しい」の正体は時間の量ではなく配分の構造
  - 自分では「集中している」つもりでも、客観的には47秒で中断している
  - 記録しなければ、問題が見えない

### 3. TimeCrowdを使った実践（30分）

- TimeCrowdでの時間記録の始め方（デモ）
  - 新規ユーザー向け: アカウント作成→最初の記録までのウォークスルー
  - 既存ユーザー向け: より精度の高い記録のコツ（カテゴリ設計、記録の粒度）
- 「ドラッカーの4-6週間記録」をTimeCrowdで実践する方法
  - まずは1週間から始める
  - 記録すべきカテゴリの設計指針
- ライブデモ: 和田自身のTimeCrowdデータを見せながら解説
- 次回までの宿題: 「2週間、全業務時間をTimeCrowdで記録してきてください」

### 4. Q&A / まとめ（10分）

- 第1回のまとめ: 「まず記録する。記憶ではなく記録で」
- 次回予告: 「第2回では、記録したデータを使って"何に時間を浪費しているか"を整理します。特に会議に注目します」

## 第2回: 時間を整理する -- 会議という時間浪費に向き合う

### ドラッカーの3ステップにおける位置づけ
> 「次に、時間を管理する。非生産的な要求に対してノーと言う。成果を生まない仕事を排除する。他の人にできることは他の人に任せる」

Step 2 = 記録を見て、削れるものを削る。

### 1. ドラッカーの考え方を整理（10分）

- ドラッカーが挙げた「時間の浪費」の4つの原因
  1. システムの欠陥や先見性の欠如から生じる時間の浪費
  2. 人員過剰から生じる時間の浪費
  3. 組織の欠陥から生じる時間の浪費
  4. 情報に関する機能障害から生じる時間の浪費
- 特に「会議」についてのドラッカーの見解
  - 「会議は組織の欠陥を補うものであり、少ないほど良い」
  - 「上級管理者が時間の4分の1以上を会議に費やしているなら、組織の構造に問題がある」

### 2. 和田としての解釈（10分）

- 時間浪費の原因整理
  - 現代の会議問題: Microsoft (2024) によると会議の60%がアドホック
  - 日本企業の会議の実態（1日275回の中断）
- 会議というものに改めて向き合う
  - 「その会議、本当に必要ですか？」の3つの判定基準
    1. 議題が明確か
    2. 参加者全員が必要か
    3. 会議以外の方法で解決できないか
- ドラッカーの提言を現代に翻訳すると

### 3. TimeCrowdを使った実践（30分）

- 自社の会議時間に向き合ってみる
  - 既存ユーザー: 第1回の宿題で記録した2週間のデータから、会議時間の割合を算出する
  - 新規ユーザー: 一般的なデータをベースに自社の状況を推定する
- ライブ分析: TimeCrowdのレポート機能で会議カテゴリを抽出
  - 全体に占める会議時間の比率
  - 会議の種類別の時間（定例、アドホック、1on1等）
  - 「この会議を半分にしたら、何時間生まれるか」のシミュレーション
- ワークショップ: 「削れる会議リスト」を各自作成
  - 参加者同士でシェア（可能であれば）
- 次回までの宿題: 「"削れる会議リスト"から実際に1つ削って、2週間のデータ変化を記録してきてください」

### 4. Q&A / まとめ（10分）

- 第2回のまとめ: 「記録を見て、削れるものを削る」
- 次回予告: 「第3回では、削って生まれた時間を"まとめる"方法を学びます。予実管理の新機能もご紹介します」

## 第3回: 時間をまとめる -- 自由になる時間を設計する

### ドラッカーの3ステップにおける位置づけ
> 「最後に、自由になった時間をまとめる。大きなまとまりにする。細切れの時間は役に立たない」

Step 3 = 生まれた時間を大きなブロックに集約し、成果を生む仕事に充てる。

### 1. ドラッカーの考え方を整理（10分）

- 「細切れの時間は役に立たない」
  - ドラッカー: 30分のブロックが6つあっても、3時間のまとまった時間1つには及ばない
  - 知識労働には最低90分のまとまった時間が必要
- 「自由裁量時間」の概念
  - 記録→整理を経て生まれた時間を、意図的に設計する
  - 何に使うかを先に決め、それを守る

### 2. 和田としての解釈（10分）

- 自由になる時間をまとめる（予実の話）
  - 「時間を節約すること」と「時間を設計すること」は別の行為
  - 予実管理 = 時間の「予算」を立て、実績と比較する
  - お金に予算があるように、時間にも予算が必要
- AI時代の時間設計
  - AIで浮いた時間が「仕事の膨張」に吸収されるパラドックス（HBR 2025）
  - 意図的に設計しなければ、削った時間は消える

### 3. TimeCrowdを使った実践（30分）

- 予実管理機能の紹介（新機能）
  - 時間の「予算」を立てる方法
  - 予算vs実績のダッシュボード
  - デモ: 和田自身の予実データを見せながら解説
- 時間設計ワークショップ
  - Step 1: 自分の「自由裁量時間」を特定する（第1回・第2回のデータから）
  - Step 2: その時間を「何に使うか」を決める（Deep Workの時間、戦略思考の時間等）
  - Step 3: TimeCrowdで「時間予算」を設定する
  - Step 4: 来週から実践し、予実を追跡する
- 参加者同士での共有（時間設計の発表）

### 4. Q&A / まとめ / シリーズ全体の締め（10分）

- 3回のまとめ
  - 第1回: 時間を記録する（まず現状を知る）
  - 第2回: 時間を整理する（浪費を削る）
  - 第3回: 時間をまとめる（生まれた時間を設計する）
- ドラッカーの言葉で締める
  - 「時間は最も希少な資源であり、時間を管理できなければ、何も管理できない」
- 「削減された時間を何に使うかは、使う人間が決める」
