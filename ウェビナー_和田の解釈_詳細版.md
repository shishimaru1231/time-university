# ドラッカー×和田の解釈 -- ウェビナーカリキュラム詳細

原典: P.F.ドラッカー『経営者の条件』第2章「汝の時間を知れ」（上田惇生訳）

---

## ドラッカー第2章の構造

ドラッカーは第2章を以下の流れで構成している:

```
1. 時間についての実際の認識
   「時間こそ真に普遍的な制約条件である」

2. 時間の使い方を知る
   「記憶ではなく記録によって」

3. 時間の使い方を診断する
   「しなくてもよいことは何か」を問う

4. 時間浪費の原因を整理する
   組織構造の欠陥が時間を奪う

5. 自由になる時間をまとめる
   「細切れの時間は役に立たない」
```

---

## 第1回: 時間を記録する

### ドラッカーの原文から

**時間についての実際の認識:**
- 成果をあげる者は時間からスタートする。計画からスタートしない
- 時間が何にとられているかを明らかにすることからスタートする
- 時間は他のもので代替できない。お金は借りられ、人は雇えるが、時間だけは借りることも雇うことも買うこともできない
- 時間の供給は硬直的で、需要がいかに大きくとも供給は増加しない

**時間の使い方を知る:**
- ほとんどの人が自分の時間の使い方を知っていると思っている。だが実際に記録してみると、まったく違うことがわかる
- 記憶に頼ってはいけない。リアルタイムで記録する
- 少なくとも年に2回、3〜4週間にわたって記録する

### 和田の解釈（第1回で語る内容）

**「時間は唯一平等な資源」の意味を掘り下げる:**

ドラッカーは「時間は借りることも雇うこともできない」と書いた。これは1966年の洞察だが、2026年の今、この言葉はさらに重い意味を持つ。

お金がない会社には投資家がいる。人がいない会社には採用がある。技術がない会社にはAIがある。しかし時間がない会社には、時間を供給してくれる外部が存在しない。

時間は「管理すべきリソース」ではなく、「すべてのリソースの前提条件」である。お金も人もあるのに成果が出ない会社は、時間の使い方に問題がある。

**「記憶ではなく記録で」が実践されていない現実:**

ドラッカーは「記憶に頼るな」と書いた。58年前の話だ。しかし今も、ほとんどのビジネスパーソンは自分の時間の使い方を「感覚」で把握している。

感覚がどれだけ当てにならないかを示すデータがある:
- Gloria Mark (UCI, 2023): 人は自分では「集中している」と思っているが、実際には平均47秒で注意が途切れている。自己中断が全中断の49%
- つまり「自分は集中して仕事をした」という記憶は、客観的なデータと大きくずれている

ドラッカーが「記録せよ」と言った理由はここにある。人間の時間認識は構造的に不正確だから。

**TimeCrowdとドラッカーの接続:**

ドラッカーは「3〜4週間リアルタイムで記録する」と書いた。1966年にはそれは紙と鉛筆でやるしかなかった。記録の負担が大きすぎて、多くの人が実践できなかった。

TimeCrowdはドラッカーの提言を「1クリックで実践できる仕組み」にしたツールだ。ドラッカーの思想が58年かけてようやくテクノロジーで実装された。

今日のワークショップでTimeCrowdを使って記録を始めていただく。それは「ツールの使い方を覚える」のではなく、「ドラッカーが58年前に提言したことを、ようやく実践する」ということだ。

## 第2回: 時間を整理する

### ドラッカーの原文から

**時間の使い方を診断する:**
- 記録を手にしたら、次の問いを投げかける:
  - 「まったくしなかったならば、何が起こるか」→ 何も起こらないなら、その仕事はやめる
  - 「他の人でもやれることはないか」→ あるなら任せる
  - 「自分が他の人の時間を浪費していないか」

**時間浪費の原因を整理する:**
- ドラッカーが挙げた4つの原因:
  1. システムの欠陥や先見性の欠如から繰り返し起こる問題 → 同じ危機が繰り返されるのはマネジメントの怠慢
  2. 人員過剰 → 人が多すぎると調整に時間を取られる
  3. 組織の欠陥 → 会議が多すぎるのは組織構造の問題
  4. 情報に関する機能障害 → 必要な情報が適切な形で届かない

**会議について:**
- 「会議は組織の欠陥を補完するものである」
- 「理想的な組織とは、会議のない組織である」
- 「上級管理者が時間の4分の1以上を会議に費やすならば、組織の構造に問題がある」
- 会議が多いということは、本来1人の人間の仕事であるべきものが、複数の人間に分散しているということ

### 和田の解釈（第2回で語る内容）

**「しなかったならば、何が起こるか」のインパクト:**

この問いは簡単に見えるが、実際にやってみると衝撃的な結果が出る。

TimeCrowdのデータを見てみると、多くのビジネスパーソンの時間の中に「やめても何も起こらない仕事」が少なからず存在する。定例報告、形骸化したレビュー、誰も読まない週報。これらは「昔からやっているから」続いているだけで、成果に貢献していない。

ドラッカーはこれを「明日行うべきことを決めるのではない。明日行うために今日何をなすべきかを決める」と表現した。「やること」ではなく「やめること」を決めるのが、時間設計の第一歩。

**「会議は組織の欠陥を補完するもの」を現代に翻訳する:**

ドラッカーは「理想的な組織には会議がない」と書いた。これは会議不要論ではない。会議が必要になるのは、1人で判断すべきことが組織設計の問題で複数人に分散しているから、という構造的な指摘だ。

現代のデータを見ると:
- Microsoft (2024): ナレッジワーカーは1日に275回の中断を受けている。会議の60%がアドホック
- Shangguan (2025): チーム生産性の向上の60%は「個人の能力向上」ではなく「チーム内の時間再配分」による

つまり、会議を減らすだけでは解決しない。会議が必要になっている「組織の欠陥」を直す必要がある。

ドラッカーが指摘した4つの原因を現代語に翻訳すると:
1. 同じ問題の繰り返し → ドキュメント化・自動化がされていない
2. 人員過剰 → SlackやTeamsで「念のためCC」が増え、調整コストが爆発
3. 組織の欠陥 → 権限委譲が不十分で、すべてが会議での合議になる
4. 情報の機能障害 → 必要な情報が散在し、確認のための会議が生まれる

**「自社の会議時間に向き合う」ワークの意味:**

今日のワークでは、第1回の宿題で記録した自分のTimeCrowdデータから会議時間を抽出していただく。

見てほしいのは3つ:
1. 全業務時間に占める会議の割合（ドラッカーの基準: 25%以上なら組織に問題あり）
2. 定例 vs アドホックの比率（アドホックが多いほど組織設計が弱い）
3. 「しなかったならば何が起こるか」に対して「何も起こらない」と答えられる会議の数

この3つが見えるだけで、何を変えるべきかが明確になる。

## 第3回: 時間をまとめる

### ドラッカーの原文から

**自由になる時間をまとめる:**
- 記録→診断→整理を経て、自由裁量の時間が生まれる
- この時間を大きなまとまりに集約する
- 「細切れの時間は役に立たない」
- 半日以上のまとまった時間がなければ、重要な仕事はできない
- 「小さな時間は役に立たない。大きなまとまりの時間が必要である」
- 自由裁量時間を守るために、予備の時間を組み込んでおく必要がある

### 和田の解釈（第3回で語る内容）

**「細切れの時間は役に立たない」の科学的裏付け:**

ドラッカーは経験からこの結論に至ったが、現代の認知科学がその正しさを証明している。

Sophie Leroy (2009) の「注意残余」研究: タスクAからタスクBに切り替えると、Aへの注意が残り続ける。タスクが未完了のまま切り替えた場合、その残余はさらに大きい。30分のブロックが6つあっても、3時間のまとまった時間1つに及ばないのは、切り替えのたびに注意残余が蓄積するからだ。

ドラッカーは「半日以上のまとまった時間」と書いた。Cal Newport (2016) は同じことを「Deep Work」と名付けた。やっていることは同じだ。ドラッカーが50年先を行っていた。

**「予実管理」こそドラッカーの3ステップの完成形:**

ドラッカーは「自由裁量時間を守るために、予備の時間を組み込んでおく必要がある」と書いた。これは予算管理の発想そのものだ。

お金の管理には「予算」がある。予算を立て、実績と比較し、差異を分析し、次の予算に反映する。これが予実管理だ。

では、なぜ時間には予算がないのか。

お金には予算を立てるのに、時間には予算を立てない。これは矛盾している。ドラッカー自身が「時間はもっとも希少な資源」と書いた。最も希少な資源に予算がないのはおかしい。

TimeCrowdの予実管理機能は、この矛盾を解消する。時間に「予算」を立て、実績と比較する。ドラッカーが書いた「記録→整理→まとめる」の3ステップの先にある「設計→実行→検証→改善」のサイクルを回す仕組みだ。

**「時間を節約する」と「時間を設計する」の決定的な違い:**

ここが最も重要なポイントになる。

多くの時間管理術は「時間を節約する」ことを目的としている。効率化、時短、ライフハック。しかしドラッカーは「時間を節約せよ」とは書いていない。「自由になった時間をまとめよ」と書いた。

節約は手段であって目的ではない。目的は「まとまった時間で、重要な仕事をする」こと。

HBR (2025) の調査が示したように、AIで時間を節約しても、節約された時間は「もっと多くの仕事」に吸収される。節約した時間をどう使うかを意図的に設計しなければ、節約は無意味になる。

これが「時間を節約すること」と「時間を設計すること」の違いだ。

ドラッカーの3ステップは「節約」のための手法ではない。「設計」のための手法だ。

1. 記録する → 現状を知る（設計の前提）
2. 整理する → 無駄を削る（設計の余白を作る）
3. まとめる → 意図的に配分する（設計そのもの）

TimeCrowdの予実管理は、この3ステップの第3段階「設計する」を仕組みに落とし込んだものだ。

## 全3回を貫く和田のメッセージ

### 中心テーマ
> **ドラッカーは58年前に答えを書いていた。私たちはまだ実践できていない。**

### 各回のキーメッセージ

| 回 | ドラッカーの教え | 和田の解釈 |
|----|---------------|----------|
| 第1回 | 記憶ではなく記録で | 人間の時間認識は構造的に不正確だ。だからツールで記録する |
| 第2回 | しなかったならば何が起こるか | 会議の多さは個人の問題ではなく組織設計の問題だ |
| 第3回 | 細切れの時間は役に立たない | 「時間を節約する」と「時間を設計する」はまったく別の行為だ |

### 締めの言葉（第3回のクロージング）

ドラッカーは第2章の冒頭でこう書いた。「成果をあげる者は仕事からスタートしない。時間からスタートする。」

58年前のこの言葉が、今日のウェビナーの出発点だった。

3回にわたって、記録すること、整理すること、まとめることをやってきた。TimeCrowdというツールで、ドラッカーが紙と鉛筆でやっていたことをデジタルに実践してきた。

最後に、ドラッカーが書かなかった問いを1つ。

> **「削減された時間を何に使うかは、使う人間が決める。」**

ドラッカーは「まとめよ」と書いた。まとめた先で何をするかは、あなた自身が決めること。

時間の予算を立て、意図的に設計してください。それが、ドラッカーの3ステップの完成形です。
